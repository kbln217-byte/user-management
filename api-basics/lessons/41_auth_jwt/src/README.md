# JWTに入れてよい情報（10個）
### 基本的に漏れても重大なセキュリティ問題にならない情報
例）
```txt
| No | 情報               | 理由                    |
| -- | ---------------- | --------------------- |
| 1  | `userId` / `sub` | ユーザー識別用               |
| 2  | `role`           | 権限判定（admin / user など） |
| 3  | `scope`          | APIアクセス権              |
| 4  | `iat`            | 発行時間                  |
| 5  | `exp`            | 有効期限                  |
| 6  | `iss`            | トークン発行者               |
| 7  | `aud`            | 対象サービス                |
| 8  | `tenantId`       | マルチテナント識別             |
| 9  | `permissions`    | 細かい権限                 |
| 10 | `featureFlags`   | 機能フラグ                 |
```

## JWTに入れてはいけない情報
## 絶対に入れてはいけない情報(セキュリティ的に)

例）
```txt
| No | 情報             | 理由          |
| -- | -------------- | ----------- |
| 1  | password       | 即アカウント乗っ取り  |
| 2  | passwordHash   | オフライン解析の危険  |
| 3  | クレジットカード番号     | PCI違反       |
| 4  | セッションID        | セッションハイジャック |
| 5  | API secret key | サービス侵害      |
| 6  | private key    | 完全破壊        |
| 7  | OTP / 2FAコード   | 認証突破        |
| 8  | 個人情報（住所・電話）    | プライバシー問題    |
| 9  | refresh token  | トークン再発行可能   |
| 10 | 内部DB ID体系      | 情報漏洩        |
```

<!-- 401/403/400/404/500 のエラー例をREADMEにまとめる（どういう時に起きるか） -->

401 Unauthorized	アクセス権が無い、または認証に失敗	
403 Forbidden	閲覧権限が無いファイルやフォルダ	
400 Bad Request	一般的なクライアントエラー
404 Not Found	Webページが見つからない
500 Internal Server Error	何らかのサーバ内で起きたエラー
